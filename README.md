끄글 · 끝말잇기 검색 엔진(https://singrum.github.io/ggeugle)
=======================
끄글이란?
--------
끝말잇기에서 승리, 패배, 순환 단어 정보를 동적으로 제공하는 웹 어플리케이션입니다. 


룰 설정
-----
룰 설정 아코디언 버튼을 클릭하면 끝말잇기 룰을 설정할 수 있습니다. 사전, 두음법칙, 품사, 글자수, 인덱스를 각각 설정하여 룰을 커스터마이징 할 수 있습니다.

예시 : 구엜룰
>사전 : (구)표국대  
>두음법칙 : 표준  
>품사 : 명사  
>글자수 : 전체 선택  
>인덱스 : 앞에서1, 뒤에서1  

예시 : 쿵쿵따
>사전 : (구)표국대  
>두음법칙 : 표준  
>품사 : 명사  
>글자수 : 3  
>인덱스 : 앞에서1, 뒤에서1  

예시 : 앞말잇기
>사전 : (구)표국대  
>두음법칙 : 표준  
>품사 : 명사  
>글자수 : 전체 선택  
>인덱스 : 뒤에서1, 앞에서1  




승리음절, 패배음절, 순환음절
--------------------------
체르멜로 정리에 의하면 끝말잇기에서는 모든 턴에 누구에게 필승 전략이 존재하는지가 이미 정해져 있습니다. 그러나 누구인지를 알아내는 알고리즘은 시간 복잡도가 굉장히 커서 실제로 구현할 수 없습니다. 그것의 대안으로 도입한 개념이 승리음절, 패배음절, 순환음절입니다.
끝말잇기에선 기본적으로 전에 썼던 단어를 다시 쓰지 못하지만, 무수히 다시 쓸 수 있다는 가정을 도입할 수 있습니다. 이러한 가정을 도입했을 때, 어떤 음절 x에 대해서 다음과 같이 상황을 나눌 수 있습니다.
1. x가 주어진 플레이어에게 필승 전략이 존재한다.
2. x가 주어진 플레이어의 상대방에게 필승 전략이 존재한다.
3. x가 주어진 플레이어와 상대방 모두에게 필승 전략이 존재하지 않는다.


x가 1을 만족시키는 경우 x를 **승리음절**이라고 하고, 2를 만족시키는 경우 x를 **패배음절**이라고 하고, 3을 만족시키는 경우 x를 **순환음절**이라고 합니다.
일반적으로 승리음절을 받으면 기본 룰에서도 필승 전략이 존재하고, 패배음절을 받으면 기본 룰에서 상대방에게 필승 전략이 존재합니다. 순환음절의 경우 기본 룰에서 필승 전략이 존재하는지 여부는 알기 힘듭니다.


검색
----
음절을 클릭하거나 검색창에 음절을 입력하면, 그것으로 시작하는 단어들 중 유의미한 단어들의 리스트를 보여줍니다. 유의미하다는 것은 음절이 승리음절일 때 필승 전략에 해당하는 단어, 패배음절일 때 그것으로 시작하는 모든 단어, 순환음절일 때 또 다른 순환음절로 끝나는 단어들을 의미합니다.
